# Contributing to @beltiston/events

We welcome contributions! Please follow these guidelines to help maintain quality and consistency.

## Development Setup
Fork the repository and clone your fork.

## Commit Guidelines
Use clear, conventional commit messages. For example:

fix: correct listener cleanup

feat: add once listener support

refactor: improve wildcard resolution

## Pull Requests
Keep PRs focused and concise.

Add or update tests for all changes.

Document any new APIs in README.md.

## Code Style
Prefer descriptive variable and function names.

Proper indentation
